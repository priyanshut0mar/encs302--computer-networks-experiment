## Key Concepts Demonstrated

### ARP (Address Resolution Protocol)
- Before ICMP can run, ARP resolves the destination IP тЖТ MAC address.
- First ping triggers an ARP **broadcast** (`FF:FF:FF:FF:FF:FF`).
- ARP reply is **unicast** back to the requester.

### ICMP Ping
- Only begins after ARP completes.
- Echo Request (Type 8) тЖТ Echo Reply (Type 0).

### Switch MAC Learning
- Switch learns source MACs from incoming frames.
- Unknown destination тЖТ frame is **flooded** to all ports.
- Known destination тЖТ frame is **unicast forwarded** to specific port.
- Entries are **DYNAMIC** with a 300-second aging timer (default).
